This page describes how to manage clean rooms, including how to:

* Update a clean room owner and comment
* Add, remove, and edit data assets and notebooks
* Grant access to a clean room
* Monitor clean room activity
* Delete a clean room

These tasks can be performed by all collaborators in a clean room.

## Before you begin[​](#before-you-begin "Direct link to Before you begin")

Managing clean rooms requires the following permissions, depending on the task:

* To view a clean room in the clean room list or to view clean room details, you must be the owner of the clean room, a metastore admin, or have one of the following privileges on the clean room: `MANAGE`, `MODIFY CLEAN ROOM`, `EXECUTE CLEAN ROOM TASK`, or `BROWSE`.
* To update the owner of a clean room, you must be the owner of the clean room, a metastore admin, or have the `MANAGE` privilege on the clean room. The creator is automatically assigned as the owner of the clean room in their Databricks account. The collaborator organization's metastore admin is automatically assigned ownership of the clean room in their Databricks account. You can transfer ownership. See [Manage object ownership](/aws/en/data-governance/unity-catalog/manage-privileges/#manage-ownership).
* To add and remove data assets and notebooks in a clean room and to update a comment, you must be the owner of the clean room or have the `MODIFY CLEAN ROOM` privilege on the clean room.
* To grant access to a clean room, you must be the owner, a metastore admin, or have the `MANAGE` privilege on the clean room.
* To delete a clean room, you must be the owner or have the `MANAGE` privilege on the clean room.

## Update a clean room[​](#update-a-clean-room "Direct link to Update a clean room")

1. In your Databricks workspace, click  **Catalog**.
2. Click the **Clean Rooms >** button.

   Alternatively, click the  gear icon at the top of the **Catalog** pane and select **Clean Rooms**.
3. Select the clean room from the list.
4. Make any of the following updates:

   * **Edit comment**: Click the  edit icon next to the comment. Comment changes apply only to the clean room securable in your Unity Catalog metastore. It does not get propagated to other collaborators.
   * **Transfer owner**: Click the  edit icon next to the **Owner** name.
   * **Add tables, volumes, and views**: See [Step 3. Add data assets and notebooks to the clean room](/aws/en/clean-rooms/create-clean-room#add-assets).
   * **Remove tables, volumes, and views**: Click the  kebab menu (also known as the three-dot menu) on the asset row and select **Remove data asset**.
   * **Add notebooks**: See [Step 3. Add data assets and notebooks to the clean room](/aws/en/clean-rooms/create-clean-room#add-assets).
   * **Update notebooks**: Click the  kebab menu on the notebook row and select **Update notebook**. In the dialog, browse for and select the updated notebook.

     Make any updates directly in the notebook before you select it in the Clean Rooms interface.
   * **Remove notebooks**: Click the  kebab menu on the notebook row and select **Delete notebook**.

## Grant access to a clean room[​](#grant-access-to-a-clean-room "Direct link to Grant access to a clean room")

Here's a breakdown of clean room ownership and privilege assignment:

* The clean room owner has all privileges on the clean room.
* The clean room owner, metastore admin, and users with manage privilege can grant other principals access to the clean room.
* The creator is automatically assigned as the owner of the clean room in their Databricks account.
* The collaborator organization's metastore admin is automatically assigned ownership of the clean room in their Databricks account.
* You can transfer ownership. See [Manage object ownership](/aws/en/data-governance/unity-catalog/manage-privileges/#manage-ownership).

The owner, metastore admin, and users with the `MANAGE` privilege can grant the following privileges on a clean room:

* `BROWSE`, which lets you list the clean room and view details like comments, owner, assets, and run history.
* `MODIFY CLEAN ROOM`, which lets you do everything that `BROWSE` grants, plus add and remove data assets, add and remove notebooks, and update comments.
* `EXECUTE CLEAN ROOM TASK`, which lets you do everything that `BROWSE` grants, plus run notebooks in a clean room.
* `MANAGE`, which lets you do everything that `BROWSE` grants, plus grant permissions to yourself or other users, and change the owner.

To grant a principal privileges on a clean room:

1. In your Databricks workspace, click  **Catalog**.
2. At the top of the **Catalog** pane, click the  gear icon and select **Clean Rooms**.

   Alternatively, from the **Quick access** page, click the **Clean Rooms >** button.
3. Select the clean room from the list.
4. Go to the **Permissions** tab.
5. Select the user, group, or service principal, click **Grant**, and select the privileges you want to grant.

For more information about granting privileges, see [Manage privileges in Unity Catalog](/aws/en/data-governance/unity-catalog/manage-privileges/).

## Enable SecureConnect for a clean room[​](#enable-secureconnect-for-a-clean-room "Direct link to enable-secureconnect-for-a-clean-room")

Preview

This feature is in [Public Preview](/aws/en/release-notes/release-types).

SecureConnect lets a clean room provider share assets from storage behind a firewall or private endpoint without adding each collaborator's network to an allowlist. Databricks brokers the connection through a managed proxy. For an overview, requirements, and provider firewall setup, see [Share data behind a firewall with SecureConnect](/aws/en/opensharing/secureconnect-provider).

When SecureConnect is enabled for clean rooms, it applies to all the assets you share into clean rooms. It does not affect assets that other collaborators share. You cannot enable it per clean room.

Before you enable SecureConnect, complete the provider firewall configuration described in [Step 1: Configure your storage firewall](/aws/en/opensharing/secureconnect-provider#step-1). Collaborators on classic compute and open recipients must allowlist Databricks control plane IPs for the provider's region.

To enable SecureConnect on your clean rooms:

1. In your Databricks workspace, click  **Catalog**.
2. At the top of the **Catalog** pane, click the  gear icon and select **Clean Rooms**.

   Alternatively, in the upper-right side, click **Share > Clean Rooms**.
3. In the upper-right, turn on **Enable SecureConnect**.

## Monitor clean room notebook runs[​](#monitor-clean-room-notebook-runs "Direct link to monitor-clean-room-notebook-runs")

All notebook runs from all collaborators are logged on the **Runs** tab in the Clean Rooms UI. You can filter runs by status and collaborator who ran the task.

1. In your Databricks workspace, click  **Catalog**.
2. Click the **Clean Rooms >** button.

   Alternatively, click the  gear icon at the top of the **Catalog** pane and select **Clean Rooms**.
3. Select the clean room from the list.
4. Go to the **Runs** tab.

You can also view runs for a specific notebook by selecting the notebook in the clean room UI and going to the **Runs** tab on the notebook details page.

## Monitor clean room logs[​](#monitor-clean-room-logs "Direct link to Monitor clean room logs")

The `clean_room_events` system table logs all clean-room related events in your Databricks metastore and enables you to view all actions that you and other collaborators took on your clean rooms.

To learn how to use this system table to gain insight into clean room activities, see [Clean room events system table reference](/aws/en/admin/system-tables/clean-rooms).

## Delete a clean room[​](#delete-a-clean-room "Direct link to Delete a clean room")

When you delete a clean room, collaborators are unable to use it, but it appears on their list of clean rooms until they delete the clean room object on their side. When you delete a clean room, running tasks are canceled immediately.

1. In your Databricks workspace, click  **Catalog**.
2. Click the **Clean Rooms >** button.

   Alternatively, click the  gear icon at the top of the **Catalog** pane and select **Clean Rooms**.
3. Select the clean room from the list.
4. Click the  kebab menu on the upper right corner of the page.
5. Select **Delete**.
6. Confirm that you want to delete the clean room, and click **Delete**.

## Troubleshoot clean room issues[​](#troubleshoot-clean-room-issues "Direct link to Troubleshoot clean room issues")

You may encounter the following error messages or issues when you work with clean rooms.

**Other collaborators have left the clean room, so the clean room is no longer usable. Please delete this clean room securable.**

In a no-trust clean room, any collaborator can delete the central clean room. This error message means that the other collaborator deleted the central clean room, and your reference to the central clean room is no longer valid. You can only delete the clean room object in your Databricks workspace. You cannot use it.

**Collaborator X already has a clean room named Y**

No collaborator can change the clean room name. This ensures that all collaborators can reference the clean room name without ambiguity. This error means that the other collaborator already has a clean room with the same name as the one you chose. Choose another clean room name.

**Please accept the Serverless terms of service before using Clean Rooms**

Serverless compute is required for central clean rooms. You do not need to enable serverless compute in your own workspace to use clean rooms. But an account admin needs to accept the serverless compute terms of service. If you need assistance, contact your Databricks representative.
