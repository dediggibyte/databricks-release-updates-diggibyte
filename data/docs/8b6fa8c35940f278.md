The **partner-powered AI features** setting controls whether some features are powered by models hosted by a partner provider. By default, this setting is enabled for all workspaces that do not have a [compliance security profile (CSP)](/aws/en/security/privacy/enhanced-security-compliance) enabled. For CSP workspaces, this setting is disabled by default.

Enabling this setting gives users access to more features. When enabled, features that use this setting are currently powered by Azure OpenAI service, OpenAI on Databricks, or Anthropic on Databricks. This setting can be disabled in either the [account](#account) or the [workspace](#workspace).

When the **partner-powered AI features** setting is disabled, AI assistive features use Databricks-hosted models where available for workspaces in the Americas and Europe. For workspaces in other geographies, Databricks-hosted models require [cross geography processing](/aws/en/resources/databricks-geos#enable-cross-geo-processing). Some features, such as Genie Code's agentic capabilities, are not available when the **partner-powered AI features** setting is disabled. Genie Code is still available without its agentic capabilities, using a Databricks-hosted model.

## For an account: Disable or enable partner-powered AI features[​](#for-an-account-disable-or-enable-partner-powered-ai-features "Direct link to for-an-account-disable-or-enable-partner-powered-ai-features")

The **partner-powered AI features** setting is enabled by default. However, account admins can disable the setting for all workspaces within an account. Account admins can also enforce the setting for the entire account.

Enabling or disabling partner-powered AI features for your account is captured as an account event in audit logs. See [Account-level authentication events](/aws/en/admin/account-settings/audit-logs#account-accounts).

1. As an account admin, log in to the [account console](https://accounts.cloud.databricks.com).
2. Click **Settings**.
3. Click the **Feature enablement** tab.
4. For the **Enable partner-powered AI features** option, select **Off** or **On**.
5. Prevent workspace overrides with the **Enforce** setting:

If you select **Off** for the **Enable partner-powered AI features** option, Databricks-hosted models continue to power AI assistive features when available.

## For a workspace: Disable or enable partner-powered AI features[​](#for-a-workspace-disable-or-enable-partner-powered-ai-features "Direct link to for-a-workspace-disable-or-enable-partner-powered-ai-features")

If your account admin enables partner-powered AI features including Genie Code, it is enabled by default in workspaces. Unless the account admin chose to enforce the setting, workspace admins can disable it for their workspaces.

To disable partner-powered AI features in a workspace:

1. Click your username in the top bar of the Databricks workspace > **Settings**.
2. In the left pane under **Workspace admin**, click the **Advanced** tab.
3. To disable partner-powered AI features, click the toggle **Off** for **Partner-powered AI features**.

If you select **Off** for the **partner-powered AI features** option, Databricks-hosted models continue to power AI assistive features when available.
