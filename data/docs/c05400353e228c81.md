Governance Hub is a centralized, account-level UI for monitoring governance across your Databricks account. It provides a unified view of data health and coverage, AI usage and spend, and cost drivers, with prioritized insights and recommendations. All views respect existing permissions.

Beta

This feature is in [Beta](/aws/en/release-notes/release-types). An account admin enables it from the account console **Previews** page. See [Manage Databricks previews](/aws/en/admin/workspace-settings/manage-previews).

Governance Hub is in the account console. After [enabling Governance Hub](#enable), administrators can open the account console by clicking **Select workspace** at the top of the workspace UI and selecting **Manage account**, or by going directly to the account console URL for their cloud:

<https://accounts.cloud.databricks.com>

In the account console, the **Data**, **AI**, and **Cost** pages that comprise Governance Hub appear under **Governance** in the left sidebar.

## Governance Hub pages

Governance Hub includes the following pages:

* + [Data](/aws/en/admin/governance-hub/data)
  + An overview of your data estate, including object access, asset usage, governed tags, data classification, and data quality monitoring.
* + [AI](/aws/en/admin/governance-hub/ai)
  + An overview of your AI usage and spend, including token usage, budgets, and model and user activity.
* + [Cost](/aws/en/admin/governance-hub/cost)
  + An overview of your entire Databricks spend, including cost drivers, budgets, tagging coverage, and cost recommendations.

## Enable Governance Hub

An account admin enables the preview from the account console **Previews** page. See [Manage Databricks previews](/aws/en/admin/workspace-settings/manage-previews). Data might take up to a day to load after you enable the preview.

## Access and permissions

Governance Hub does not introduce new permissions, nor does it manage access to data or resources. Users see Governance Hub pages based on the access they already have, which is reflected by their role:

| Role | Access |
| --- | --- |
| [Account admin](/aws/en/admin/admin-concepts#account-admins) | Full access to the Data, AI, and Cost pages, including account-wide metrics, all workspace-level drill-downs for cost and AI, and all metastore-level drill-downs for data. |
| [Workspace admin](/aws/en/admin/admin-concepts#workspace-admins) | The Cost and AI pages for their assigned workspaces, including the ability to create and manage budgets scoped to those workspaces. |
| [Metastore admin](/aws/en/data-governance/unity-catalog/manage-privileges/admin-privileges#metastore-admins) | The Data page for their assigned metastores. |
