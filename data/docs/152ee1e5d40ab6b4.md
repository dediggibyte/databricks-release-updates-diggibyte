Beta

This feature is in [Beta](/aws/en/release-notes/release-types). Workspace admins can control access to this feature from the **Previews** page. See [Manage Databricks previews](/aws/en/admin/workspace-settings/manage-previews).

The managed Slack Access and Integration Logs connector in Lakeflow Connect allows you to ingest workspace access logs and app integration change logs from Slack into Databricks.

## Feature availability[​](#feature-availability "Direct link to feature-availability")

| Feature | Availability |
| --- | --- |
| UI-based pipeline authoring | Not supported |
| API-based pipeline authoring | Supported |
| Declarative Automation Bundles | Supported |
| Incremental ingestion | Supported |
| Unity Catalog governance | Supported |
| Orchestration using Databricks Workflows | Supported |
| SCD type 2 | Not supported  Slack access and integration logs are append-only. |
| Automated schema evolution: New and deleted columns | Supported |
| Automated schema evolution: Data type changes | Not supported |
| Automated schema evolution: Column renames | Not supported  Requires a full refresh. |

## Authentication methods[​](#authentication-methods "Direct link to Authentication methods")

| Authentication method | Availability |
| --- | --- |
| OAuth U2M | Supported |
| OAuth M2M | Not supported |
| Basic authentication (username/password) | Not supported |

## Ingest from Slack Access and Integration Logs in 3 steps[​](#ingest-from-slack-access-and-integration-logs-in-3-steps "Direct link to ingest-from-slack-access-and-integration-logs-in-3-steps")

note

Before starting, review the [Databricks user persona](/aws/en/connect/managed-ingestion), [supported interfaces](/aws/en/ingestion/lakeflow-connect/faq#interfaces), [ingestion frequency](/aws/en/ldp/pipeline-mode), and [common patterns](/aws/en/ingestion/lakeflow-connect/common-patterns).

## Start ingesting from Slack Access and Integration Logs[​](#start-ingesting-from-slack-access-and-integration-logs "Direct link to Start ingesting from Slack Access and Integration Logs")

1. **[Configure Slack for ingestion](/aws/en/ingestion/lakeflow-connect/slack-access-integration-logs-source-setup)** (Admins) — Set up Slack to authenticate with Databricks.
2. **[Create a Unity Catalog connection](/aws/en/ingestion/lakeflow-connect/slack-access-integration-logs-connection)** (Admins) — Create a connection in Catalog Explorer so non-admins can create pipelines.
3. **[Create an ingestion pipeline](/aws/en/ingestion/lakeflow-connect/slack-access-integration-logs-pipeline)** (Admins or non-admins) — Select any supported interface and create a pipeline from an existing connection.
