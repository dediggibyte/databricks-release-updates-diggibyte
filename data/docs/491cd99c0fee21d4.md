Beta

Omnigent is in [Beta](/aws/en/release-notes/release-types). For the open-source documentation, see the [Omnigent documentation](https://omnigent.ai/).

Omnigent provides a common layer over Claude Code, Codex, Cursor, Pi, and the agents you write yourself: swap or combine harnesses without rewriting, keep them in check with policies and sandboxing, and collaborate in real time on the same live session, from any device. Define an agent in a short YAML file, then swap one line to change its harness or model while your tools, prompts, skills, and policies stay the same.

Databricks provides a fully managed version of Omnigent, including:

* A Databricks-operated Omnigent server that integrates with your workspace's identity provider.
* Model access through the [Foundation Model APIs](/aws/en/machine-learning/foundation-model-apis/) and [AI Gateway](/aws/en/ai-gateway/).

* Databricks Sandboxes for secure, collaborative agent coding and knowledge work. Databricks Sandbox is available in [select regions on AWS](/aws/en/compute/serverless/sandbox#limitations).

Omnigent is an open source project. For complete documentation, including concepts, harnesses, custom agents, interfaces, policies, and sandboxes, see the [Omnigent documentation](https://omnigent.ai/).

## Get started[​](#get-started "Direct link to Get started")

To set up Omnigent against your workspace and launch your first agent, see the [Omnigent quickstart](/aws/en/omnigent/quickstart).

## Availability[​](#availability "Direct link to Availability")

* Omnigent requires the **Omnigent** preview enabled for your workspace. See [Manage workspace-level previews](/aws/en/admin/workspace-settings/manage-previews#workspace).
* Omnigent requires a workspace in a region that supports Databricks Unity AI Gateway. See [Databricks feature availability by region](/aws/en/resources/feature-region-support).

* To launch Omnigent sessions with a [Databricks Sandbox](/aws/en/compute/serverless/sandbox), you must also have:
  + A workspace in a region that also supports Databricks Sandbox. See [Limitations](/aws/en/compute/serverless/sandbox#limitations).
  + The **Sandbox** preview enabled for your workspace. See [Manage workspace-level previews](/aws/en/admin/workspace-settings/manage-previews#workspace).

## Limitations[​](#limitations "Direct link to Limitations")

* Only built-in contextual policies are supported. Apply them from the workspace UI or by declaring them in your agent configuration. [Custom YAML-based policies](https://omnigent.ai/docs/policies/overview) are not available.
* Native Windows support is not available. For now, run Omnigent inside [WSL2](https://learn.microsoft.com/windows/wsl/install) (Windows Subsystem for Linux) and follow the Linux instructions from your WSL2 distribution.

* A Databricks Sandbox host always routes model access through AI Gateway. You cannot bring your own model API keys for a Databricks Sandbox host.
* Databricks Sandbox does not integrate with serverless egress controls. Do not use Databricks Sandbox in a workspace with serverless egress control (SEG) enabled. For more information, see [What is serverless egress control?](/aws/en/security/network/serverless-network-security/network-policies).
