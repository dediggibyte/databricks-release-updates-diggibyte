This feature is in [Beta](/aws/en/release-notes/release-types). Workspace admins can control access to this feature from the **Previews** page. See [Manage Databricks previews](/aws/en/admin/workspace-settings/manage-previews).

Lakehouse Federation can sync table and column comments from supported external data sources to foreign tables in Unity Catalog. Synced comments appear in table metadata, including Catalog Explorer, `DESCRIBE`, and `DESCRIBE EXTENDED` output.

## Requirements[​](#requirements "Direct link to Requirements")

* **Compute**: Use Databricks Runtime 18.2 or above, or a SQL warehouse on a channel that runs Databricks SQL version 2026.15 or above. See [SQL warehouse channels](/aws/en/sql/release-notes/#channels).

## Supported data sources[​](#supported-data-sources "Direct link to Supported data sources")

Foreign table comments are synced for the following data sources:

* [AWS Glue metastore](/aws/en/query-federation/hms-federation-glue)

* [Google BigQuery](/aws/en/query-federation/bigquery)
* [Hive metastore federation](/aws/en/query-federation/hms-federation-concepts)
* [MySQL](/aws/en/query-federation/mysql)
* [Oracle](/aws/en/query-federation/oracle) on Databricks Runtime 18.3 or above
* [PostgreSQL](/aws/en/query-federation/postgresql)
* [Amazon Redshift](/aws/en/query-federation/redshift)
* [Snowflake query federation](/aws/en/query-federation/snowflake) and [Snowflake catalog federation](/aws/en/query-federation/snowflake-catalog-federation)
* [Teradata](/aws/en/query-federation/teradata)

Foreign table comments are not synced for the following data sources:

* [Azure Synapse (SQL Data Warehouse)](/aws/en/query-federation/sqldw)
* [Databricks-to-Databricks query federation](/aws/en/query-federation/databricks)
* [Microsoft SQL Server](/aws/en/query-federation/sql-server)
* [Salesforce Data 360](/aws/en/query-federation/salesforce-data-cloud) and [Salesforce Data 360 file sharing](/aws/en/query-federation/salesforce-data-cloud-file-sharing)

## Sync comments from the external source[​](#sync-comments-from-the-external-source "Direct link to Sync comments from the external source")

For supported data sources, Databricks fetches foreign table comments when it resolves foreign table metadata at query time. When comments change in the external source, the updated comments are synced the next time a query resolves the foreign table.

To sync comments without running a query against the table data, run `DESCRIBE EXTENDED` on the foreign table:

SQL

```
DESCRIBE EXTENDED <catalog-name>.<schema-name>.<table-name>;
```
