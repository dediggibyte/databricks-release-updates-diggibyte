# Select rows to ingest with Lakeflow Connect

Row filtering lets an ingestion pipeline pull only the rows that match a
condition, rather than replicating an entire source table. You express the
condition much like a SQL `WHERE` clause. Filtering runs on the source side of
the connector, so unmatched rows are never transferred, staged, or stored.

## How it works

- You define a row filter per source object (table, stream, or object) as a
  boolean expression over that object's columns.
- The filter is applied during the initial load and on every subsequent
  incremental update, so the target stays consistent with the filter over time.
- Because filtering happens before transfer, it reduces data movement, storage,
  and downstream compute compared with ingesting everything and filtering later.

## Supported connectors

Row filtering is supported for the following SaaS connectors:

- Google Analytics
- Salesforce
- ServiceNow

It is also supported for all query-based connectors, where the filter is pushed
into the source query.

## Prerequisites

- A Lakeflow Connect ingestion pipeline on a supported connector.
- Unity Catalog enabled for the destination catalog and schema.
- Permission to edit the ingestion pipeline definition.

## Limitations

- Filters reference only columns available on the source object; joins across
  objects are not supported in a filter.
- Changing a filter after the initial load can require a full refresh of the
  affected object to remain consistent.
- Non-deterministic source functions are not supported inside a filter.

## Configure a row filter

1. Open your Lakeflow Connect ingestion pipeline.
2. Select the source object you want to filter.
3. Add a row filter condition using the object's columns.
4. Validate the filter against a sample of source rows.
5. Run the pipeline; the filter applies to the initial load and all incremental
   updates.

## Use cases

- Ingest only rows for a specific region, business unit, or active-record flag.
- Exclude test or archived records from a SaaS system.
- Reduce cost and duplication for very large source tables.
